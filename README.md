1. 引言
--------
1.1 编写目的
本说明书旨在详细描述《进化贪吃蛇》游戏软件的需求分析、系统设计、模块划分及实现细节，为开发人员、测试人员和维护人员提供完整的参考文档，确保软件质量与可维护性。

1.2 项目背景
传统贪吃蛇游戏玩法单一，本项目在经典玩法基础上融入用户账户系统、角色成长、多种敌人、环境交互、技能树、日夜循环等元素，旨在提供更丰富的游戏体验。软件采用 Python 语言与 Pygame 库开发，面向单机用户。

1.3 适用范围
本文档适用于《进化贪吃蛇》软件生命周期内的需求分析、设计、编码、测试及后期维护阶段。

1.4 术语定义
- FPS：每秒帧数 (Frames Per Second)
- Verlet 积分：一种用于模拟粒子运动的数值积分方法
- 粒子系统：用于实现爆炸、烟雾等视觉效果的技术
- HUD：平视显示器 (Heads-Up Display)，游戏中显示状态信息的部分
- JSON：JavaScript 对象表示法，用于数据交换


2. 系统概述
------------
2.1 系统目标
- 实现用户注册与登录，区分不同用户的游戏记录。
- 提供流畅的贪吃蛇操控体验，支持键盘移动、加速。
- 包含成长系统：通过吃食物获得分数和经验，升级解锁技能。
- 构建丰富的敌人类型和环境危险物，增加挑战性。
- 提供技能树供玩家自定义能力。
- 记录每局游戏的详细日志，便于查看历史成绩。
- 通过粒子特效、屏幕震动、日夜循环等增强视觉表现。

2.2 运行环境
- 操作系统：Windows 7/10/11、Linux、macOS
- Python 版本：3.8 及以上
- 依赖库：pygame (可使用 pip install pygame 安装)
- 分辨率：窗口固定 1200×800 像素
- 硬件要求：普通个人计算机，支持 60 FPS 图形渲染

2.3 系统主要功能
- 用户注册与登录
- 游戏主循环：蛇移动、食物收集、碰撞检测
- 敌人 AI 生成与行为
- 墙壁、岩浆、传送门等地图元素
- 技能树（九头蛇、泰坦、相位转移、磁铁）
- 日夜循环光照效果
- 游戏日志记录与查看
- 迷你地图与 HUD 信息显示


3. 功能需求
------------
3.1 用户管理
  (1) 注册新用户：输入用户名（2~20字符）和密码（≥3字符），密码使用 SHA-256 哈希存储。
  (2) 用户登录：验证用户名和密码，匹配后进入游戏。
  (3) 游戏日志：每次游戏结束自动记录：用户ID、用户名、开始时间、持续时间（秒）、得分。
  (4) 日志查看：主菜单或游戏中按 F5 可查看全部日志，支持上下翻页。

3.2 游戏核心
  (1) 蛇的控制：W/A/S/D 或方向键控制移动方向；按住空格键消耗体力加速。
  (2) 蛇身物理：头部带动身体，采用 Verlet 积分方式实现柔顺的跟随效果。
  (3) 食物生成：地图内随机位置生成 20 个食物，食物具有随机颜色和经验值（10~30）。
  (4) 吃食物判定：蛇头与食物距离小于 30 像素时视为吃掉，分数增加，经验增加，蛇身加长 3 节，并在该位置生成新的食物。
  (5) 生命与体力：蛇有生命值（初始 100），碰撞敌人或危险物受伤；体力自动回复，加速时快速消耗。
  (6) 等级与经验：经验达到阈值升级，提升最大生命值，获得 1 技能点。
  (7) 游戏结束条件：生命值 ≤ 0 或碰撞非泰坦状态下的墙壁。
  (8) 游戏数据重置：游戏结束或重新开始将初始化蛇状态、地图、敌人等。

3.3 敌人系统
  (1) 猎人：向蛇匀速移动，接触造成伤害。
  (2) 炮塔：固定位置，每 2 秒发射一枚射弹，射弹对蛇造成伤害。
  (3) 幽灵：半透明、不规则运动，接触伤害。
  (4) Boss：当分数达到 500 的倍数时生成，具有高生命值，能发射环形弹幕，自身长有分段身体。
  (5) 敌人管理：敌人随时间自动生成，生成间隔随分数增加缩短；射弹在击中蛇或超过寿命后消失。

3.4 环境危险物
  (1) 墙壁：矩形障碍物，碰撞即游戏结束（泰坦技能可免疫）。
  (2) 岩浆：圆形区域，持续对蛇造成小额伤害。
  (3) 传送门：进入后将蛇传送到对应的出口，并产生粒子特效。

3.5 技能树
  玩家消耗技能点可学习以下技能：
  - 九头蛇：增加额外蛇头（视觉效果）。
  - 泰坦：增大体型，可无视墙壁碰撞。
  - 相位转移：临时无敌状态（10秒），可升级 3 次。
  - 磁铁：自动吸引 200 像素范围内的食物。

3.6 界面与交互
  - 主菜单提供注册、登录、查看日志选项。
  - 游戏中 HUD 显示当前用户、分数、等级、蛇长、生命条、经验条、体力条。
  - 迷你地图显示蛇、食物、敌人、墙壁的全局位置。
  - 按 T 打开技能树界面，按 P 暂停游戏，按 F5 查看游戏日志（游戏中）。
  - 游戏结束界面显示最终得分，破纪录时显示祝贺语，可按 R 重新开始或 ESC 返回主菜单。

3.7 特效与增强
  - 粒子效果：吃食物、碰撞、传送时产生彩色粒子。
  - 屏幕震动：受伤或吃食物时产生随机偏移。
  - 日夜循环：游戏时间周期性变化，夜晚时屏幕变暗，蛇周围有光照范围。
  - 食物脉动动画、敌人及 Boss 的视觉细节。


4. 非功能需求
--------------
4.1 性能需求
  - 稳定维持 60 FPS，不发生明显掉帧。
  - 粒子、敌人、射弹等对象数量上限合理，避免内存泄漏。

4.2 可用性需求
  - 操作键位符合常见游戏习惯，提供按键提示。
  - 中文界面，字体清晰。
  - 输入框支持 TAB 切换、退格、回车确认。

4.3 可靠性需求
  - 用户数据和游戏日志使用 JSON 文件本地持久化，写入异常时不影响游戏运行。
  - 游戏退出时（包括强制关闭）尽可能保存当前游戏记录。

4.4 可维护性需求
  - 模块化设计，类职责单一，便于扩展新敌人、新技能。
  - 命名规范，关键算法添加注释。


5. 系统架构设计
----------------
5.1 总体架构
采用面向对象的“游戏循环”架构，主要分为：
- 数据层：UserDataManager 负责用户和日志的持久化。
- 逻辑层：Game、Snake、EnemyManager、HazardManager、ParticleSystem 等管理游戏状态与更新。
- 表现层：各类 draw 方法渲染画面。
- 控制层：事件处理函数分发用户输入。

游戏主循环依次执行：处理事件 → 更新逻辑 → 绘制画面。

5.2 状态管理
使用 GameState 枚举定义游戏状态，实现有限状态机：
  MENU → REGISTER / LOGIN / USER_LOG
  REGISTER / LOGIN → PLAYING
  PLAYING → PAUSED / SKILL_TREE / GAME_OVER / USER_LOG
  PAUSED → PLAYING / MENU
  SKILL_TREE → PLAYING
  GAME_OVER → PLAYING (重启) / MENU
  USER_LOG → MENU / PLAYING (按返回来源)

5.3 模块划分
  - 工具函数模块：lerp, dist, normalize, hash_password。
  - 数据管理模块：UserDataManager。
  - 实体模块：Particle, Snake, Food, Enemy 子类, Projectile, Wall, Lava, Portal。
  - 系统管理模块：ParticleSystem, EnemyManager, HazardManager。
  - UI 模块：InputBox, Game 类中的绘制函数。
  - 主控模块：Game 类（包含主循环）。


6. 详细设计
------------
6.1 用户数据管理模块 (UserDataManager)
  属性：
    - users: Dict[str, dict] 用户名到用户信息的映射
    - logs: List[dict] 游戏日志列表
  关键方法：
    - _load() / _save_users() / _save_logs(): 文件读写与异常处理
    - register(username, password) -> (bool, msg): 验证输入，创建用户，写入文件
    - login(username, password) -> (bool, msg): 校验密码
    - add_log(username, start_time, duration, score): 添加日志条目
    - get_logs() -> List[dict]: 获取所有日志

6.2 游戏世界模块
6.2.1 蛇类 (Snake)
  属性：
    - x, y: 蛇头位置
    - vx, vy: 当前速度
    - angle: 朝向角度
    - speed: 基础速度
    - body: List[BodySegment] 身体节段
    - health, stamina: 生命与体力
    - boost, invincible, titanic, magnet: 状态标志
    - heads: 额外头数量（影响显示）
  主要方法：
    - reset(): 初始化到地图中央，生成 20 节身体
    - update(dt, keys, game): 处理输入、移动、体力、无敌计时、身体跟随、磁铁效果
    - grow(segments): 在尾部添加指定数量节段
    - draw(): 绘制蛇身、蛇头、眼睛；无敌时颜色变化；支持泰坦体型

6.2.2 食物 (Food)
  属性：x, y, radius, pulse, color, xp_value
  方法：update(dt) 用于脉动动画；draw() 绘制并带缩放效果

6.2.3 粒子系统 (ParticleSystem)
  使用 Particle 数据类存储粒子状态。
  方法：
    - emit(): 生成一批粒子，参数化数量、颜色、速度范围、大小、生命
    - update(dt): 更新粒子位置、速度（含重力）、生命，移除死亡粒子
    - draw(surface, cx, cy): 仅绘制摄像机视野内的粒子，支持透明度渐变

6.2.4 敌人系统
  基类 Enemy 定义共同属性（x, y, health, damage, speed）和抽象方法 update, draw。
  - Hunter: 向蛇直线移动。
  - Turret: 定时计算朝向蛇的角度并发射 Projectile。
  - Ghost: 正弦扰动移动，带半透明绘制。
  - Boss: 较大体型，多节身体，环形弹幕，带血条。
  Projectile 类：带有速度、伤害和生命期。
  EnemyManager 类：
    - 管理 enemies 列表和 projectiles 列表。
    - spawn_timer 控制敌人生成间隔，随分数缩短。
    - update(dt, game): 更新所有敌人和射弹，进行碰撞检测，触发粒子与屏幕震动。

6.2.5 地图与危险物
  - Wall: 矩形障碍物，draw 时转换到屏幕坐标。
  - Lava: 圆形，绘制时脉动效果，update 进行相位动画，check 中计算持续伤害。
  - Portal: 圆形入口，检测到传送时瞬间移动蛇并生成粒子。
  HazardManager 类：
    - 在 generate() 中随机生成 30 个墙壁、10 个岩浆、3 对传送门。
    - update(dt) 更新岩浆和传送门的动画。
    - draw() 调用各自绘制。
    - check(snake, game): 检测蛇与墙壁碰撞（泰坦除外）、岩浆伤害、传送触发，返回是否致命碰撞。

6.3 界面与渲染模块
  主要渲染方法在 Game 类中：
  - _draw_menu(): 主菜单界面
  - _draw_auth_screen(): 注册/登录公用界面
  - _draw_world(): 游戏场景绘制：网格背景 → 危险物 → 食物 → 敌人/射弹 → 蛇 → 粒子 → 夜间遮罩
  - _draw_hud(): 用户提示条、生命/经验/体力条、迷你地图、操作提示
  - _draw_skill_tree(): 半透明背景，显示技能列表及解锁状态
  - _draw_gameover(): 游戏结束画面
  - _draw_log_screen(): 日志表格，支持滚动

  摄像机跟随蛇头移动，带平滑插值，屏幕震动通过随机偏移实现。

6.4 输入处理模块
  Game 类的 handle_events() 方法根据当前状态分发事件到具体的 _ev_xxx 方法。
  InputBox 类处理文本输入，支持焦点切换和字符过滤。

6.5 主循环与游戏管理
  Game 类的 run() 方法构成无限循环，每帧调用 handle_events(), update(), draw()。
  update() 中仅在 PLAYING 状态下更新游戏逻辑，否则仅更新粒子系统和消息计时。
  游戏开始时调用 start_new_game() 重置所有变量并生成地图与食物。


7. 数据设计
------------
7.1 用户数据
  文件：snake_users.json
  格式示例：
  {
    "player1": {
      "id": "1",
      "username": "player1",
      "password": "hashed_password"
    }
  }

7.2 游戏日志数据
  文件：snake_game_logs.json
  格式：列表，每项为：
  {
    "id": "1",
    "username": "player1",
    "start_time": "2026-05-20 15:30:00",
    "duration_sec": 120.5,
    "score": 850
  }

7.3 游戏运行时数据
  游戏状态由 Game 对象的属性维护，包括蛇、敌人、食物列表、技能状态、分数等，均为内存数据，不持久化。


8. 界面设计
------------

  - 右上角迷你地图（150×150），缩放显示全局元素
  - 底部提示文字：“按F5显示游戏用户日志”，“T:技能 P:暂停 ESC:菜单”

8.4 技能树界面
  - 半透明黑色背景遮罩
  - 显示“技能树”标题和当前技能点数量
  - 列出四种技能，已解锁显示绿色，未解锁灰色，并附说明
  - 按数字键 1~4 学习对应技能，按 T 或 ESC 关闭

8.5 游戏结束界面
  - 半透明遮罩，显示“游戏结束”和最终分数
  - 若打破历史最高分显示“新纪录”
  - 操作提示：R 重新开始，ESC 返回菜单

8.6 日志查看界面
  - 表格形式，列：ID、用户名、开始时间、持续时长(秒)、得分
  - 支持键盘上下键和鼠标滚轮翻页
  - 底部提示返回方式（根据入口不同显示不同按键）


9. 测试计划
------------
9.1 单元测试
  - 对工具函数 (lerp, dist, normalize, hash_password) 进行边界值测试。
  - 对 UserDataManager 的注册/登录逻辑进行用例测试（空输入、重复用户、错误密码等）。
  - 对 Snake 的 grow 方法验证身体长度变化。

9.2 功能测试
  - 注册登录流程、日志记录与查看。
  - 蛇移动、加速、吃食物升级。
  - 敌人 AI 行为（猎人追踪、炮塔射击、幽灵移动、Boss 弹幕）。
  - 墙壁碰撞、岩浆伤害、传送门传送。
  - 技能树学习及效果验证（泰坦撞墙、相位无敌、磁铁吸引）。
  - 状态切换（暂停、技能树、游戏结束、日志）的正确性。
  - 日夜循环光照效果是否平滑。

9.3 性能测试
  - 长时间运行（>30分钟）观察帧率是否稳定在 60 FPS。
  - 大量粒子、敌人存在时内存和 CPU 占用是否异常。
  - JSON 文件频繁读写是否导致卡顿（目前仅在关键点写入，风险较低）。


10. 部署与运行
---------------
10.1 安装依赖
  确保已安装 Python 3.8+，执行：
    pip install pygame

10.2 运行游戏
